# Angel Beats! -1st beat- 重编译解包汉化
本汉化版是在【死了没法儿忍汉化组】的旧版汉化基础上，通过采用对《Angel Beats! -1st beat-》的原始源代码重新编译的方式，进行的新版汉化。


# 为什么要重新汉化？
旧版本的汉化是基于ALKA Translation英化组的英文版翻译而来，在二次翻译的过程中，存在相当数量的语义偏差；同时，旧版汉化是在编译后的脚本上直接修改UTF-8字符串，无法删除或添加已有的注音标注等控制符，给最终的阅读体验带来了诸多不便。

如今，生成式人工智能正在飞速发展。以ChatGPT、DeepSeek等为代表的大语言模型，在短短的两三年内，就从提供情绪价值的玩具，变成了强而有力的程序员或是翻译官。虽然本次重新汉化并没有简单地采用AI工作流进行全面汉化，但即使仅仅只是将AI作为解释词义或是句意的字典，都远胜在原版汉化时期可用的Weblio等日语词典网站。针对网络用于和流行语的翻译，以及对各种梗的解释尤为如此。

本次汉化基于原始日文实体光盘版重新进行翻译，在修正错误翻译的同时，通过添加注音标注等方式，改善中文环境下的阅读体验。

# 重编译是如何进行的？
Siglus引擎的`Scene.pck`剧本在生成过程中会进行编译，将UTF-8字符串以外的全部代码和注释全部编译处理，无法直接以人类可读的方式进行编辑。但引擎同时会将原始的未编译脚本文件打散，原模原样地保存在`Scene.pck`里，并可使用特定的工具重新还原，实现源码级别的重新翻译与开发。

早期的重编译依赖于从Sekai Project流出的Siglus引擎开发套件。该套件版本较为陈旧，已无法进行《anemoi》等新作品加入的手柄支持代码等新功能的编译；且使用内部渠道流出的该套件进行二次汉化存在法律风险。

本次使用的重编译脚本来自于民间爱好者自行开发的 [siglus-ssu](https://github.com/Jirehlov/SiglusSceneScriptUtility/) Python工具链，实现对`Scene.pck`中未编译脚本文件的提取。关于该工具及其配套Visual Studio Code插件的使用，可以查看原作者仓库内的介绍。

# 相比旧版本汉化，本次汉化有哪些技术层面的改动？

本次新版本汉化使用的加密密钥为原光盘版`Scene.pck`的密钥，与旧版本汉化使用的基于ALKA英化组版本的`Scene.pck`文件密钥不同。

所有的人名翻译，均通过将文本映射挂载到Gameexe.ini的方式实现，不对原本的姓名进行改动。
